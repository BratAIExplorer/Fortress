### PHASE 2: ALLOCATION LAYERS (Weeks 3-4, June 2026)

**Objective:** Build 4-layer allocation engine, replace hardcoded 10X Moonshot

**Tasks:**

1. **Create Allocation Layer Schema**
   ```sql
   CREATE TABLE allocation_layers (
     id UUID,
     name VARCHAR (CORE | GROWTH | AGGRESSIVE | SATELLITE),
     risk_level INT (1-5),
     target_return_pct FLOAT,
     description TEXT,
     created_at TIMESTAMP
   );
   
   CREATE TABLE allocation_components (
     id UUID,
     layer_id UUID,
     etf_ticker VARCHAR,
     allocation_pct FLOAT,
     rebalance_frequency VARCHAR (quarterly | semi-annual),
     FOREIGN KEY (layer_id) REFERENCES allocation_layers(id)
   );
   ```

2. **Populate 4 Allocation Layers**

   - SATELLITE: DBC 2%, VNQ 2%, IBIT 1% (optional)

3. **Update Investment Genie**
   - Replace hardcoded allocation logic
   - Query allocation_layers based on risk profile (Conservative/Balanced/Growth/Aggressive)
   - Return percentages from database, not hardcoded values

4. **Build Allocation Comparison UI**
   - Show user: "Your allocation vs. CORE layer vs. GROWTH layer vs. Top 20 optimization"
   - Highlight missing exposure (e.g., "You have 0% bonds — consider +5% BND")
